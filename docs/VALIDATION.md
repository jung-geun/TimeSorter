# TimeSorter — 교차 검증 결과 및 분석

## 개요

학습된 DPO 모델의 출력을 **gpt-5.5**가 독립적으로 생성한 기준 답안과 비교하여 품질을 평가합니다.

### 검증 방식 (2-Phase)

```
Phase 1: gpt-5.5가 원본 이메일 5건만 보고 독립적으로 태스크 추출 + 우선순위 결정
              ↓
Phase 2: [모델 출력] vs [gpt-5.5 기준] 비교 채점 (태스크 커버리지 / 우선순위 / 추론 품질)
```

독립 생성 후 비교하는 방식으로 앵커링 편향을 방지합니다.

## 강점

- **형식 일관성**: `오전/오후 구분 + 항목별 시간 배치` 형식을 일관되게 출력
- **범주 포괄**: 5개 이메일 모두에서 핵심 업무 범주는 추출 (계약서, 보고서, 미팅, 점심, 코드 리뷰)
- **의존 태스크 분리 시도**: 보고서 작성과 참조 메일 발송을 별도 항목으로 분리

## 약점

- **날짜 파싱 오류**: "5/24"를 "오늘"로, "5/24 오전"을 "오늘 오전 12시"로 잘못 해석
- **환각 생성**: 존재하지 않는 "오전 10시 회의" 추가 → 실제 오후 2시 미팅 누락으로 이어짐
- **선행 의존 관계 미인식**: "참석 가능 여부 확인 → 미팅 준비 → 미팅 참석" 3단계 의존 구조 파악 실패
- **담당자 정보 탈락**: 수신자명·연락처 같은 실행 필수 정보가 스케줄에서 제거됨

## 개선 방향 (v0.6 계획)

| 문제 | 개선 방안 |
|------|-----------|
| 날짜 파싱 오류 | 프롬프트에 오늘 날짜 명시 + 날짜 정규화 후처리 |
| 환각 미팅 | 이메일 → 태스크 추출 시 시간 정보 명시 강제 |
| 의존 관계 | 의존성 체인 SFT 예제 추가 (선행/후행 레이블) |
| 마감 앞당김 | DPO rejected 예제에 마감일 오류 유형 추가 |
| 담당자 누락 | 스케줄 출력 형식에 `(담당자: __)` 필드 포함 |
